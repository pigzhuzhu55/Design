#静态代理模式写法，以及讲解

- 静态代理模式
- 为其他对象提供一种代理以控制对这个对象的访问
- 比如，一个对象不适合或者不能直接引用另外一个对象，而是通过代理对象去引用，代理对象起到中介的作用
- 不改变原类的代码，而增强原类对象的功能(前置、后置、环绕、异常等处理逻辑)，这样好像跟装饰者模式很像，
对的，但是目的区别在于，代理模式意在代理中控制使用者对目标对象的访问，以及功能增强。
- 静态代理：手动编写的.class文件，在程序运行前就已经存在了。
